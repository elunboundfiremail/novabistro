================================================================================
                            1. INTRODUCCION
================================================================================

NovaBistro es un sistema de gestion integral para restaurantes desarrollado
con tecnologias web modernas. El sistema permite la administracion de pedidos,
mesas, personal, productos y categorias mediante una API REST robusta y
escalable.

El proyecto fue desarrollado utilizando FastAPI como framework backend,
PostgreSQL como sistema gestor de base de datos, y siguiendo las mejores
practicas de desarrollo de software.

El sistema esta diseñado para facilitar la gestion operativa de un restaurante,
permitiendo a los meseros tomar pedidos desde tablets, actualizar el estado
de las mesas en tiempo real, y proporcionar a la cocina una vista actualizada
de todos los pedidos pendientes.

================================================================================
                            3. ALCANCE DEL PROYECTO
================================================================================

El sistema NovaBistro incluye las siguientes funcionalidades:

MODULO DE ROLES:
- Gestion de roles de usuario (Administrador, Mesero, Cocinero, Cajero)
- Asignacion de permisos por rol

MODULO DE PERSONAL:
- Registro de empleados con datos personales (CI, nombre, apellidos)
- Asignacion de roles al personal
- Control de personal activo/inactivo

MODULO DE MESAS:
- Registro de mesas con numero, capacidad y ubicacion
- Control de estado de mesas (disponible, ocupada, reservada)
- Organizacion por zonas (Salon principal, Terraza, Salon VIP)

MODULO DE CATEGORIAS:
- Clasificacion de productos del menu
- Gestion de categorias de comida (Entradas, Platos principales, Bebidas,
  Postres, Sopas)

MODULO DE PRODUCTOS:
- Registro de productos con nombre, descripcion y precio en Bolivianos (Bs)
- Control de disponibilidad de productos
- Asignacion a categorias

MODULO DE PEDIDOS:
- Creacion de pedidos por mesa
- Asignacion de personal responsable
- Control de estados (pendiente, en preparacion, listo, entregado)
- Calculo automatico de totales
- Registro de observaciones

MODULO DE DETALLE DE PEDIDOS:
- Registro de productos por pedido
- Control de cantidades
- Calculo de subtotales
- Observaciones especificas por producto

================================================================================
                            4. DESCRIPCION DEL SISTEMA
================================================================================

NovaBistro es un sistema diseñado especificamente para restaurantes en Bolivia,
considerando las caracteristicas propias del contexto local:

- Uso de Carnet de Identidad (CI) para identificacion de personal
- Precios en Bolivianos (Bs)
- Nomenclatura sin tildes para compatibilidad de sistemas
- Estructura organizacional tipica de restaurantes bolivianos

FLUJO OPERATIVO:

1. El personal (meseros) toma pedidos desde tablets o dispositivos moviles
2. Los pedidos se registran en el sistema asociados a una mesa especifica
3. La cocina visualiza en tiempo real todos los pedidos pendientes
4. El personal de cocina actualiza el estado de los pedidos
5. Los meseros son notificados cuando los pedidos estan listos
6. Se actualiza el estado de la mesa al completar el servicio

VENTAJAS DEL SISTEMA:

- Eliminacion de errores por anotaciones manuales
- Comunicacion directa entre sala y cocina
- Control de tiempos de preparacion
- Historial completo de pedidos
- Reportes y estadisticas en tiempo real
- Gestion eficiente de inventario

================================================================================
                        6. BASE DE DATOS
================================================================================

6.1. MODELO ENTIDAD-RELACION

El sistema cuenta con 7 tablas principales:

ROLES (1) ----< (N) PERSONAL (1) ----< (N) PEDIDOS
                                              |
CATEGORIAS (1) ----< (N) PRODUCTOS           |
                            |                 |
                            |                 V
                            +--------> DETALLE_PEDIDOS
                                       
MESAS (1) ----< (N) PEDIDOS

RELACIONES:

- Un ROL puede tener muchos PERSONAL
- Un PERSONAL puede atender muchos PEDIDOS
- Una MESA puede tener muchos PEDIDOS
- Una CATEGORIA puede tener muchos PRODUCTOS
- Un PEDIDO puede tener muchos DETALLE_PEDIDOS
- Un PRODUCTO puede estar en muchos DETALLE_PEDIDOS

6.2. DESCRIPCION DE TABLAS

TABLA: roles
Descripcion: Almacena los diferentes roles de usuario del sistema
Campos:
- id_rol (INTEGER, PK, IDENTITY): Identificador unico del rol
- nombre (VARCHAR(50), UNIQUE, NOT NULL): Nombre del rol
- descripcion (TEXT): Descripcion detallada del rol
- activo (BOOLEAN, DEFAULT TRUE): Estado del rol

TABLA: personal
Descripcion: Informacion de los empleados del restaurante
Campos:
- id_personal (INTEGER, PK, IDENTITY): Identificador unico
- ci (VARCHAR(20), UNIQUE, NOT NULL): Carnet de Identidad
- nombre (VARCHAR(100), NOT NULL): Nombre del empleado
- apellido_paterno (VARCHAR(100), NOT NULL): Apellido paterno
- apellido_materno (VARCHAR(100)): Apellido materno
- fecha_nacimiento (DATE): Fecha de nacimiento
- direccion (TEXT): Direccion del domicilio
- telefono (VARCHAR(20)): Numero de telefono
- email (VARCHAR(100), UNIQUE): Correo electronico
- id_rol (INTEGER, FK): Referencia a roles
- activo (BOOLEAN, DEFAULT TRUE): Estado del empleado

TABLA: mesas
Descripcion: Informacion de las mesas del restaurante
Campos:
- id_mesa (INTEGER, PK, IDENTITY): Identificador unico
- numero (INTEGER, UNIQUE, NOT NULL): Numero de mesa
- capacidad (INTEGER, NOT NULL): Cantidad de personas
- ubicacion (VARCHAR(50)): Zona o ubicacion
- estado (VARCHAR(20), DEFAULT 'disponible'): Estado actual
- activo (BOOLEAN, DEFAULT TRUE): Estado de la mesa

TABLA: categorias
Descripcion: Clasificacion de productos del menu
Campos:
- id_categoria (INTEGER, PK, IDENTITY): Identificador unico
- nombre (VARCHAR(100), UNIQUE, NOT NULL): Nombre de la categoria
- descripcion (TEXT): Descripcion de la categoria
- activo (BOOLEAN, DEFAULT TRUE): Estado de la categoria

TABLA: productos
Descripcion: Productos disponibles en el menu
Campos:
- id_producto (INTEGER, PK, IDENTITY): Identificador unico
- nombre (VARCHAR(150), NOT NULL): Nombre del producto
- descripcion (TEXT): Descripcion del producto
- precio_bs (DECIMAL(10,2), NOT NULL): Precio en Bolivianos
- id_categoria (INTEGER, FK): Referencia a categorias
- disponible (BOOLEAN, DEFAULT TRUE): Disponibilidad
- activo (BOOLEAN, DEFAULT TRUE): Estado del producto

TABLA: pedidos
Descripcion: Registro de pedidos realizados
Campos:
- id_pedido (INTEGER, PK, IDENTITY): Identificador unico
- numero_pedido (VARCHAR(20), UNIQUE, NOT NULL): Numero de orden
- id_mesa (INTEGER, FK): Referencia a mesas
- id_personal (INTEGER, FK): Referencia a personal
- fecha_pedido (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP): Fecha y hora
- estado (VARCHAR(20), DEFAULT 'pendiente'): Estado del pedido
- total_bs (DECIMAL(10,2), DEFAULT 0.00): Total en Bolivianos
- observaciones (TEXT): Observaciones generales
- activo (BOOLEAN, DEFAULT TRUE): Estado del pedido

TABLA: detalle_pedidos
Descripcion: Productos incluidos en cada pedido
Campos:
- id_detalle (INTEGER, PK, IDENTITY): Identificador unico
- id_pedido (INTEGER, FK, ON DELETE CASCADE): Referencia a pedidos
- id_producto (INTEGER, FK): Referencia a productos
- cantidad (INTEGER, NOT NULL): Cantidad solicitada
- precio_unitario_bs (DECIMAL(10,2), NOT NULL): Precio unitario
- subtotal_bs (DECIMAL(10,2), NOT NULL): Subtotal calculado
- observaciones (TEXT): Observaciones del producto

6.3. INDICES CREADOS

Para optimizar el rendimiento del sistema se crearon los siguientes indices:

- idx_pedidos_estado: Indice en pedidos(estado)
- idx_pedidos_mesa: Indice en pedidos(id_mesa)
- idx_productos_categoria: Indice en productos(id_categoria)
- idx_detalle_pedido: Indice en detalle_pedidos(id_pedido)
- idx_personal_rol: Indice en personal(id_rol)

- GET /productos - Listar todos los productos activos

- PUT /productos/{id} - Actualizar producto

- GET /pedidos - Listar todos los pedidos activos

2. Ejecutar el instalador
3. Configurar contraseña para usuario postgres: 12345678
4. Dejar puerto por defecto: 5432

8.4. CONFIGURACION DE LA BASE DE DATOS

1. Abrir terminal de PostgreSQL:
   psql -U postgres

2. Crear la base de datos:
   CREATE DATABASE novabistro;

3. Salir de psql:
   \q

4. Ejecutar el script de creacion de tablas:
   psql -U postgres -d novabistro -f base_datos\schema.sql

5. Verificar que las tablas se crearon:
   psql -U postgres -d novabistro
   \dt

8.5. INSTALACION DEL PROYECTO

3. Ejecutar la aplicacion:

4. Verificar que la API esta funcionando:
   http://127.0.0.1:8000
